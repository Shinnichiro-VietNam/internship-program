# Git Learning Assignments for Interns

Welcome! This guide will help you get comfortable with Git and GitHub. By the end, you’ll understand how to work with repositories, collaborate using branches and pull requests, and resolve conflicts.

---

## 📂 Assignments

1. **Repository Structure**
   - Explore the repository and understand where resources, templates, and tasks are located.
   - Identify the `README.md` and other documentation files.

2. **Git Workflow**
   - Clone the repository to your local machine.
   - Create and checkout your own branch.
   - Make changes, commit them, and push to the remote repository.

3. **Create a Git Manual**
   - Add a new file named `git-manual.md`.
   - Summarize Git usage and workflow in your own words.
   - Include examples of common commands:
     - `git clone`
     - `git checkout -b`
     - `git commit`
     - `git push`

4. **Pull Requests**
   - Open a pull request (PR) to merge your branch into `main`.
   - Request a review from your mentor before merging.

5. **Merge Conflicts**
   - If you encounter merge conflicts, practice resolving them.
   - Document the steps you took in your `git-manual.md`.

## ✅ Expected Results

By completing these assignments, you should be able to:

- Navigate and understand the repository structure.
- Create, commit, and push changes to a Git repository.
- Write a `git-manual.md` summarizing Git usage and workflow.
- Resolve simple merge conflicts.
- Collaborate effectively using branches and pull requests.
