# Internship Program Repository

Welcome to the **Shinnichiro Vietnam Internship Program Repository**.  
This repository hosts code developed by interns during their training period. Each intern is assigned their own branch to work on exercises, assignments, and projects. The goal is to provide a structured environment for learning, collaboration, and mentorship.

---

## 📂 Repository Structure

- **Main Branch (`main`)**
  - Contains shared resources, documentation, and templates.
  - Should remain clean and free of trainee-specific code.

- **Intern Branches**
  - Each intern has a branch named after them (e.g., `runa-akitake`, `le-hanh`, `m-michida`).
  - All training code, exercises, and projects must be committed to the intern’s branch.
  - Branches are used to track individual progress.

- **Directories**
  - `docs/` → Training materials, guidelines, and references
  - `templates/` → Code templates, project skeletons, and examples
  - `resources/` → Supporting files and useful links
  - `tasks`/ → Assigned exercises and projects for each intern

---

## 🛠 Workflow Guidelines

### 1. Clone the repository

```bash
git clone https://github.com/Shinnichiro-VietNam/internship-program.git
```

### 2. Checkout your branch

```bash
git checkout your-name
```

### 3. Work on your assignments/projects

Commit regularly with meaningful messages.

Keep your branch updated by pulling changes from main when needed:

```bash
git pull origin main
```

### 4. Push your changes

```bash
git push origin your-name
```

### 5. Request a review

When you have completed an assignment or project, create a pull request to merge your branch into main. This allows mentors to review your code and provide feedback.

## 📌Rules and Best Practices

- **Commit Messages**: Use clear and descriptive commit messages to explain the changes you made.
- **Branch Management**: Do not commit directly to main. Always work on your assigned branch and create pull requests for merging.
- **Collaboration**: Feel free to ask for help from mentors.
- **Documentation**: Keep your code well-documented. Include comments where necessary to explain complex logic.
- **Respect Deadlines**: Try to complete assignments and projects within the given time frame.
- **Code Quality**: Follow coding standards and best practices. Aim for clean, readable, and efficient code.
